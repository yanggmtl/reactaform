# API Reference

Each API page should include signature, props, return values, examples, and TypeScript types.

Recommended pages:
- `useForm`: create form instances, submit, reset, access values/errors.
- `ReactaForm` / Provider: wrap app, supply definition data, locale, handlers.
- `ReactaFormRenderer`: low-level renderer for custom layouts.
- `Field` / `useField` / Controller-like helpers: connect individual inputs.
- Form Context: access form state, translation, registry hooks.
- Validation API: register validators, run validation manually, inspect results.
- Types: see `src/core/reactaFormTypes.ts` for key type exports and extension points.
